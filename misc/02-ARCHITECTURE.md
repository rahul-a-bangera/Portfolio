## System Architecture

### Overview

The portfolio website follows a **Single Page Application (SPA)** architecture built with Angular 19, deployed as a static site on GitHub Pages with automated CI/CD via GitHub Actions.

```
???????????????????????????????????????????????????????????????

                 ?

?                      GITHUB PAGES CDN                        ?
?              (Global Content Delivery Network)               ?
???????????????????????????????????????????????????????????????
                 ?
                 ? Static Assets
                 ?
???????????????????????????????????????????????????????????????
?                    ANGULAR APPLICATION                       ?
?                     (Client-Side SPA)                        ?

```

### Architecture Principles

1. **Standalone Components**: No NgModules, pure Angular 19 architecture
2. **Single Responsibility**: Each component has one clear purpose
3. **Separation of Concerns**: UI, logic, and styles separated
4. **Mobile-First**: Responsive design from smallest to largest screens
5. **Performance**: Optimized bundle size and lazy loading
6. **Maintainability**: Clear structure and comprehensive documentation

## Routing Strategy

### Single Page Application (SPA)

The application uses **anchor-based navigation** rather than Angular Router, as it's a single-page portfolio with smooth scrolling between sections.

### Navigation Implementation

```typescript
// header.component.ts
scrollToSection(sectionId: string): void {
  const element = document.getElementById(sectionId);
  if (element) {
    element.scrollIntoView({ 
      behavior: 'smooth', 
      block: 'start' 
    });
  }
}
```

### URL Structure

```
https://rahul-a.in/                    # Landing page
https://rahul-a.in/#home              # Home section
https://rahul-a.in/#resume            # Resume section
https://rahul-a.in/#blog              # Blog section
https://rahul-a.in/#contact           # Contact section
```

### Scroll Margin Configuration

```css
/* Global scroll padding for sticky header */
html {
  scroll-behavior: smooth;
  scroll-padding-top: 80px;  /* Header height + spacing */
}

/* Individual section scroll margin */
.section {
  scroll-margin-top: 80px;
}
```

## Design Patterns

### 1. Component Pattern
- **Standalone Components**: No NgModules
- **Single Responsibility**: One purpose per component
- **Template-Driven**: UI in HTML, logic in TypeScript

### 2. State Management Pattern
- **Component-Level State**: No global state management
- **Local Variables**: State stored in component properties
- **Event Emitters**: For parent-child communication (if needed)

### 3. Styling Pattern
- **Component Styles**: Scoped CSS per component
- **Global Styles**: Common styles in `styles.css`
- **CSS Custom Properties**: Variables for theme consistency
- **BEM-like Naming**: `.component-element--modifier`

### 4. Responsive Design Pattern
- **Mobile-First**: Base styles for mobile
- **Progressive Enhancement**: Add features for larger screens
- **Breakpoint-Based**: Media queries for specific screen sizes
- **Touch-Friendly**: 44x44px minimum touch targets

## Scalability Considerations

### Current Architecture Limitations

1. **Static Content**: No backend API
2. **No Database**: All data hardcoded
3. **No User Authentication**: Public site only
4. **No Content Management**: Updates require code changes

## Security Architecture

### Current Security Measures

1. **HTTPS Enforcement**: GitHub Pages automatic SSL
2. **No Backend**: No server-side vulnerabilities
3. **XSS Protection**: Angular built-in sanitization
4. **External Links**: `rel="noopener noreferrer"`
5. **No Secrets**: No API keys or credentials in code

### Future Security Considerations

- **Backend API**: Add authentication/authorization
- **Rate Limiting**: Prevent abuse
- **CORS Configuration**: Restrict origins
- **Input Validation**: Server-side validation
- **Content Security Policy**: CSP headers
